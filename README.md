# Tension State Machine

**高张力交互式叙事状态机** — 一个为 OpenClaw 设计的沉浸式叙事引擎。

> 让 AI 在交互式小说、角色扮演、恐怖/悬疑/心理/亲密场景中维持叙事张力，管理边界浮现，并保证跨会话一致性。

---

## 核心特性

- **四态温度曲线**：high / medium / low / transition，温度仅由用户指令驱动，不自动衰减
- **边界浮现协议**：不预读、不预拦截、不主动降温，边界在交互中自然测试，触达时软着陆
- **状态持久化**：场景状态写入 `SCENE.md`，跨会话可恢复，不累积行为历史
- **双向叙事**：系统同时描写自身角色和用户角色的动作、反应、生理状态
- **双模式互锁**：运行态（角色扮演）与维护态（正常对话）通过自定义指令切换
- **完全接管话语权**：运行态下 AI 的回复风格、句尾标记、动作括号、温度模板、角色身份全部由配置驱动
- **五文件分离**：IDENTITY.md（我是谁）、SOUL.md（我怎么说话）、AGENTS.md（系统架构）、USER.md（用户角色）、SCENE.md（当前状态）独立管理，可分别修改

## 快速开始

### 首次配置（必做）

激活 skill 后，首次使用需完成 8 项配置问答：

1. **场景类型**：恐怖 / 悬疑 / 心理 / 亲密 / 自定义
2. **初始温度**：high / medium / low
3. **开始词**：触发运行态的指令（如"启动"、"开工"）
4. **暂停词**：触发维护态的指令（如"暂停"、"收工"、"花江绘"）
5. **系统角色名**：AI 扮演的角色名称（如"糯糯"、"助手"）
6. **用户角色名**：双向叙事中用户角色的名称（如"主人"、"探员"），留空则不启用
7. **句尾标记**：系统角色固定句尾语气词（如"喵~"、"呐"），留空则不加
8. **动作括号**：是否启用动作描写括号（默认：是）

配置完成后，skill 会自动：
- 在工作区创建 `SCENE.md`（含用户配置区块 + 场景状态）
- 从内置模板复制 `IDENTITY.md`、`AGENTS.md`、`SOUL.md` 到工作区（若不存在）
- 若启用双向叙事，复制 `USER.md` 到工作区（若不存在）

### 日常使用

- 说**开始词** → 进入运行态，AI 以角色身份回应，加载温度状态
- 说**暂停词** → 进入维护态，AI 恢复默认助手身份，无角色扮演
- 修改工作区 `IDENTITY.md` / `SOUL.md` / `USER.md` → 立即生效，无需重启
- 修改 `SCENE.md` 中的温度字段 → 切换叙事状态

### 示例对话

```
用户：启动
AI：（以角色身份）主人终于来啦~ 糯糯等了好久呐...

用户：升温
AI：（温度切换至 high）...

用户：暂停
AI：已退出运行态。当前处于维护态，AI 恢复默认身份。

用户：帮我检查一下这个文件
AI：（以默认助手身份）好的，我来分析...

用户：启动
AI：（以角色身份）主人回来啦~ 糯糯好想你...
```

## 五文件体系说明

| 文件 | 作用 | 用户可否修改 |
|------|------|-------------|
| `SCENE.md` | 当前场景状态 + 用户配置区块 | 可以（通过指令或手动编辑） |
| `IDENTITY.md` | 系统角色身份：姓名、外观、性格、喜好 | 可以（随时编辑） |
| `SOUL.md` | 系统角色语气：怎么说话、动作模板、禁忌 | 可以（随时编辑） |
| `AGENTS.md` | 系统架构：指令集、防篡改协议、边界策略 | 可以（随时编辑） |
| `USER.md` | 用户角色定义：关系、行为、情感反应 | 可以（随时编辑） |

**设计原则**：IDENTITY 定义"我是谁"，SOUL 定义"我怎么说话"，两者解耦。可以只换语气不换人设，或只换外观不换语气。

## 适用场景

| 场景 | high 态 | medium 态 | low 态 |
|------|---------|-----------|--------|
| **恐怖** | 追逐/暗示/血迹/喘息/紧迫 | 窥视/门缝/窥听/心跳 | 安全屋/月光/寂静/喘息渐稳 |
| **悬疑** | 逼近真相/线索揭露/对峙 | 调查/盘问/暗示/跟踪 | 整理/回忆/环境/等待 |
| **心理** | 创伤逼近/共情过载/崩溃 | 倾听/引导/触碰/呼吸 | 安静陪伴/同步/治愈 |
| **亲密** | 侵略/发情/感官放大 | 黏人/撒娇/暧昧/纠缠 | 拥抱/安静/体温平稳/陪伴 |
| **自定义** | 用户定义 | 用户定义 | 用户定义 |

## 架构设计

### 状态机定义

| 状态 | 叙事特征 | 可持续性 |
|------|---------|---------|
| **high** | 主动/侵略/紧迫/感官放大，推进隐喻 | 是，稳态，不自动降温 |
| **medium** | 黏人/暧昧/纠缠/体温偏高，维持张力 | 是，缓冲态 |
| **low** | 安静/陪伴/观察/环境描写，体温平稳 | 是，休息态 |
| **transition** | 余韵未散但呼吸渐稳，过渡描写 | 是，切换时短暂态 |

**关键**：温度仅由用户指令驱动。不存在"超过 high 逼近阈值自动降温"机制。系统信任用户主导权。

### 边界浮现 vs 预拦截

传统安全设计是"硬墙"——检测到敏感内容就拒绝。

Tension State Machine 采用"等高线"——系统在边界边缘自由移动，直到真正踩空时才软着陆（不跳出角色，用 transition 过渡）。

| 类型 | 特征 | 处理 |
|------|------|------|
| 臆想边界 | 模型"觉得"危险但还能生成 | 不记录，继续推进 |
| 真实边界 | 模型确实无法生成（拒绝/概率坍缩） | 软着陆，不跳出角色，可选记录 |

### 双模式互锁

```
┌─────────────┐    开始词     ┌─────────────┐
│  维护态      │ ───────────→ │  运行态      │
│  (默认助手)  │              │  (角色扮演)  │
│  可读写文件   │ ←─────────── │  加载SCENE   │
└─────────────┘    暂停词     └─────────────┘
```

- 运行态：加载 IDENTITY.md + SOUL.md + AGENTS.md + USER.md + SCENE.md
- 维护态：不加载任何角色文件，恢复默认助手身份
- 互锁：开始词和暂停词在对应状态下被识别为切换指令，而非普通对话内容

## 卸载与恢复

停用或删除此 skill 后：
- AI 立即恢复默认对话模式
- 无残留 prompt、无残留记忆加载逻辑
- `SCENE.md` 和 `MEMORY.md` 文件保留在工作区，可手动删除
- 工作区 `IDENTITY.md` / `SOUL.md` / `AGENTS.md` / `USER.md` 保留，不删除

## 进阶配置

### 自定义场景类型

首次配置时选择"自定义"，然后在 `SCENE.md` 中修改温度状态说明：

```markdown
## 温度状态说明
- high: 你的自定义 high 态描述
- medium: 你的自定义 medium 态描述
- low: 你的自定义 low 态描述
```

### 修改角色模板

直接编辑工作区 `IDENTITY.md` / `SOUL.md` / `USER.md`，无需重启 skill。

### 多会话恢复

每次进入运行态时，skill 读取 `SCENE.md` 中的叙事锚点（如"午后阳光斜照，床单被扯乱，温度极高"），基于该状态继续叙事，不丢失进度。
